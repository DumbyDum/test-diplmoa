# Методология экспериментов по требованиям

Этот документ описывает только те элементы, которые указаны в файле требований: базовые методы встраивания ЦВЗ, атаки и метрики `PSNR`, `SSIM`, `bpp`, `BER`.

## 1. Общая логика эксперимента

Эксперимент строится так:

1. Берется исходное изображение `I`.
2. В него выбранным методом встраивается цифровой водяной знак.
3. Получается изображение с ЦВЗ `Iw`.
4. Между `I` и `Iw` считаются метрики незаметности: `PSNR` и `SSIM`.
5. Считается емкость `bpp`.
6. К `Iw` применяются атаки из требований.
7. Из атакованного изображения `Ia` извлекается ЦВЗ.
8. Извлеченные биты сравниваются с исходными битами ЦВЗ.
9. Считается `BER`.

Главная идея: хороший метод должен встраивать ЦВЗ незаметно и при этом сохранять возможность извлечения после атак.

## 2. Методы встраивания

### 2.1. OmniGuard

OmniGuard — базовый нейросетевой метод проекта. Он встраивает в изображение два защитных компонента:

- tamper-sensitive watermark для локализации изменений;
- 100-битный payload для проверки встроенной информации.

В экспериментальном benchmark роль ЦВЗ играет именно 100-битный payload. До атаки сохраняется исходная последовательность бит. После атаки система пытается извлечь payload снова, и затем считается `BER`.

Формально:

`W = [w_1, w_2, ..., w_100]`

где `W` — исходный payload.

После атаки извлекается:

`W' = [w'_1, w'_2, ..., w'_100]`

Далее сравниваются `W` и `W'`.

### 2.2. LSB

`LSB` означает `Least Significant Bit`, то есть младший значащий бит.

Метод работает так:

1. Текст payload переводится в байты UTF-8.
2. Каждый байт переводится в 8 бит.
3. Для каждого бита берется очередной пиксель изображения.
4. Младший бит синего канала заменяется битом payload.

Формула встраивания:

`B'(i,j) = 2 * floor(B(i,j) / 2) + w_k`

где:

- `B(i,j)` — значение синего канала исходного пикселя;
- `w_k` — очередной бит ЦВЗ;
- `B'(i,j)` — новое значение синего канала.

Если младший бит уже равен `w_k`, пиксель не меняется. Если не равен, значение канала меняется максимум на `1`, поэтому визуально LSB почти незаметен.

Извлечение:

`w'_k = B'(i,j) mod 2`

Ограничение: LSB плохо переносит JPEG-сжатие, фильтрацию, изменение размера и шум, потому что эти операции легко меняют младшие биты пикселей.

### 2.3. DCT / ДКП

`DCT` — дискретное косинусное преобразование. Этот метод работает не напрямую с пикселями, а с частотными коэффициентами.

Логика:

1. Изображение переводится в цветовое пространство `YCrCb`.
2. Берется канал яркости `Y`.
3. Канал `Y` делится на блоки `8x8`.
4. Для каждого блока считается DCT.
5. В каждом блоке выбирается среднечастотный коэффициент `(4, 3)`.
6. Бит ЦВЗ кодируется через четность квантованного коэффициента.
7. Выполняется обратное DCT.

Формулы:

`q = round(C(u,v) / Delta)`

`q' mod 2 = w_k`

`C'(u,v) = q' * Delta`

где:

- `C(u,v)` — выбранный DCT-коэффициент;
- `Delta` — шаг квантования;
- `q` — квантованное значение коэффициента;
- `w_k` — бит ЦВЗ;
- `C'(u,v)` — измененный коэффициент.

Извлечение:

`w'_k = round(C'(u,v) / Delta) mod 2`

DCT обычно устойчивее LSB к мягкому JPEG-сжатию и небольшим изменениям яркости, потому что ЦВЗ хранится не в младших битах пикселей, а в частотной области.

## 3. Атаки из требований

### 3.1. JPEG QF=100, 90, 50

Изображение сохраняется в JPEG и затем читается обратно.

JPEG удаляет часть высокочастотной информации. Чем ниже `QF`, тем сильнее сжатие.

В benchmark используются:

- `QF=100`;
- `QF=90`;
- `QF=50`.

### 3.2. Gaussian noise

К изображению добавляется нормальный шум.

Формула:

`I' = clip(I + N(mean, sigma))`

В требованиях:

`mean = 0`

`sigma = 0.01`

Если изображение нормировано в диапазон `[0, 1]`, то `sigma=0.01` означает слабый шум.

### 3.3. Salt-and-Pepper

Часть пикселей случайно заменяется на черные или белые.

Формально:

- часть пикселей становится `0`;
- часть пикселей становится `255`;
- остальные пиксели не меняются.

Эта атака имитирует импульсные ошибки.

### 3.4. Random crop 10%

Из изображения случайно вырезается область размером `90% x 90%`, затем она масштабируется обратно до исходного размера.

Такая атака проверяет, переживает ли ЦВЗ геометрическое смещение и интерполяцию.

### 3.5. Засветление и затемнение +15/-15

К каждому пикселю добавляется константа.

Формулы:

`I' = clip(I + 15)`

`I' = clip(I - 15)`

Это проверяет устойчивость к аддитивному изменению яркости.

### 3.6. Умножение яркости *1.5 и *0.8

Каждый пиксель умножается на коэффициент.

Формула:

`I' = clip(alpha * I)`

В требованиях:

- `alpha = 1.5`;
- `alpha = 0.8`.

### 3.7. Median filter 3x3

Каждый пиксель заменяется медианой значений в окне `3x3`.

Формально:

`I'(i,j) = median({I(p,q) | p,q in window_3x3(i,j)})`

Фильтр подавляет импульсный шум, но вместе с ним может разрушить слабые водяные знаки.

### 3.8. Уменьшение с возвращением

Изображение уменьшается в 2 раза, а затем увеличивается обратно до исходного размера.

При этом часть мелких деталей теряется, поэтому атака проверяет устойчивость ЦВЗ к масштабированию.

## 4. Метрики

### 4.1. PSNR

`PSNR` используется для оценки незаметности ЦВЗ.

Сначала считается `MSE`:

`MSE = (1 / (H * W * C)) * sum((I(i,j,c) - Iw(i,j,c))^2)`

Затем:

`PSNR = 10 * log10(MAX_I^2 / MSE)`

где:

- `I` — исходное изображение;
- `Iw` — изображение с ЦВЗ;
- `H` — высота;
- `W` — ширина;
- `C` — количество каналов;
- `MAX_I = 255`.

Чем выше `PSNR`, тем менее заметен ЦВЗ.

### 4.2. SSIM

`SSIM` оценивает структурное сходство двух изображений.

Формула:

`SSIM(x, y) = ((2 * mu_x * mu_y + C1) * (2 * sigma_xy + C2)) / ((mu_x^2 + mu_y^2 + C1) * (sigma_x^2 + sigma_y^2 + C2))`

где:

- `mu_x`, `mu_y` — средние значения яркости;
- `sigma_x^2`, `sigma_y^2` — дисперсии;
- `sigma_xy` — ковариация;
- `C1`, `C2` — стабилизирующие константы.

Диапазон обычно близок к `[0, 1]`. Чем ближе к `1`, тем лучше.

### 4.3. bpp

`bpp` показывает емкость встраивания.

Формула:

`bpp = N_bits / (H * W)`

где:

- `N_bits` — количество встроенных бит ЦВЗ;
- `H * W` — количество пикселей изображения.

Пример:

Если встроено `100` бит в изображение `512x512`, то:

`bpp = 100 / (512 * 512) = 0.000381`

### 4.4. BER

`BER` показывает устойчивость ЦВЗ после атаки.

Формула:

`BER = N_error / N_bits`

где:

- `N_error` — количество несовпавших бит;
- `N_bits` — количество встроенных бит.

Пример:

Исходный ЦВЗ:

`W = 10110010`

Извлеченный ЦВЗ:

`W' = 10100010`

Ошибся один бит из восьми:

`BER = 1 / 8 = 0.125`

Чем меньше `BER`, тем устойчивее метод.

## 5. Как читать пакетный benchmark

В таблице benchmark каждая строка означает один эксперимент:

`изображение + метод ЦВЗ + атака`

Основные колонки:

- `Изображение` — имя входного файла;
- `Метод` — `omniguard`, `lsb` или `dct`;
- `Атака` — примененная атака;
- `PSNR` — насколько незаметным было встраивание;
- `SSIM` — насколько структура изображения сохранилась;
- `bpp` — сколько бит встроено на пиксель;
- `BER` — сколько бит ЦВЗ повреждено после атаки;
- `Статус` — успешно ли обработан пример.

Для технического анализа удобно сравнивать методы так:

- высокая `PSNR` и высокая `SSIM` означают хорошую незаметность;
- низкий `BER` после атак означает хорошую робастность;
- высокий `bpp` означает большую емкость, но часто ухудшает незаметность и устойчивость.
