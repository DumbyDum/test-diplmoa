# Справка По Интерфейсу OmniGuard 2.0

Эта страница нужна как быстрый ориентир по интерфейсу.

## 1. Вкладка "Основной сценарий"

Здесь выполняется полный цикл работы:

1. загрузка исходного изображения;
2. защита изображения;
3. редактирование выделенной области;
4. анализ результата.

### 1.1. Блок "Исходное изображение"

Сюда загружается файл, который нужно защитить.

Подходящие форматы:

- `.png`
- `.jpg`
- `.jpeg`
- `.bmp`
- `.webp`

Внутри система приводит изображение к RGB и при необходимости уменьшает слишком большую сторону.

### 1.2. Поле `Document ID`

Это человекочитаемый идентификатор изображения или документа.

Примеры:

- `diploma-demo-001`
- `report-2026-04-26`
- `passport-scan-17`

Он нужен для формирования payload и последующей проверки.

### 1.3. Кнопка "Защитить изображение"

На этом шаге в изображение встраиваются:

- tamper-sensitive watermark;
- 100-битный payload.

### 1.4. Блок "Защищенное изображение"

Показывает результат после защиты.

### 1.5. Блок "Выделите область редактирования"

Здесь кистью выделяется зона, которую нужно изменить.

### 1.6. Поле `Prompt`

Используется только если система может работать через диффузионный inpainting.

Если активен fallback OpenCV, поле почти не влияет на результат.

### 1.7. Поле "Редактор изображения"

Позволяет выбрать backend редактирования:

- `Автовыбор`
- `OpenCV Telea`
- `OpenCV Navier-Stokes`
- `Diffusers Fast`
- `Diffusers Quality`

Если нужен лучший demo-результат, выбирай `Diffusers Quality`, когда модель доступна.

### 1.8. Поле "Локальный путь / model id для diffusers"

Сюда можно указать:

- локальный путь к inpainting-модели;
- или model id Hugging Face.

### 1.9. Флажок "Разрешить загрузку модели из интернета"

Если он включен, система сможет попытаться скачать diffusers-модель, если ее нет локально.

### 1.10. Кнопка "Выполнить редактирование"

Запускает изменение выделенной области.

### 1.11. Поле "Режим локализации"

Доступны режимы:

- `Гибридный` - основной рекомендуемый режим. Использует и watermark, и сравнение с защищенной опорной версией.
- `Только watermark` - анализ только по скрытому защитному сигналу.
- `Только сравнение с опорной версией` - сравнение текущего изображения с защищенной версией.

### 1.12. Поле "Порог бинарной маски"

Чем ниже порог, тем чувствительнее маска.

- маленький порог: больше подозрительных пикселей;
- большой порог: маска строже.

### 1.13. Блок "Heatmap"

Показывает карту аномалии.

Чем ярче область, тем сильнее система считает ее измененной или подозрительной.

### 1.14. Блок "Бинарная маска"

Показывает только те пиксели, которые прошли порог.

Эту маску удобнее использовать для количественного анализа.

### 1.15. Блок "Технический отчет"

Показывает структурированный JSON:

- результаты проверки payload;
- параметры анализа;
- числовые метрики;
- информацию о reference-сравнении.

### 1.16. Блок "Интерпретация"

Это человекочитаемое объяснение:

- было ли изображение изменено;
- почему система так решила;
- какие метрики особенно важны в текущем примере.

## 2. Вкладка "Benchmark"

Benchmark нужен для автоматической оценки устойчивости системы.

Сначала изображение защищается, затем к нему применяются разные атаки, после чего считаются метрики.

Что показывает benchmark:

- насколько стабильно восстанавливается payload;
- насколько локализатор реагирует на атаки;
- насколько предсказанная маска совпадает с эталонной маской там, где она известна.

---

## 3. Вкладка "Анализ внешнего файла"

Эта вкладка нужна, если вы:

- встроили payload в изображение;
- сохранили защищенную версию;
- изменили ее вручную в стороннем редакторе;
- хотите проверить измененный файл через OmniGuard.

Что загружается:

- внешне измененное изображение;
- защищенная опорная версия;
- ожидаемый `document_id`.

Рекомендация:

- если опорная защищенная версия есть, лучше использовать `Гибридный` режим.
