# Справка по интерфейсу OmniGuard 2.0

Интерфейс разделен на пять основных вкладок. Первые четыре вкладки предназначены для работы, пятая вкладка — общий справочник с объяснениями.

## 1. Основной сценарий

Эта вкладка показывает полный цикл OmniGuard:

1. загрузка изображения;
2. встраивание payload и tamper-sensitive watermark;
3. локальное редактирование области;
4. анализ результата;
5. построение heatmap и бинарной маски.

Используй ее для рабочего сценария, в котором система защищает одно изображение и затем пытается найти изменение.

## 2. Методы встраивания ЦВЗ

Вкладка нужна для объяснения базовых методов:

- OmniGuard;
- LSB;
- DCT / ДКП.

Пользователь загружает одно изображение, вводит payload / document_id и выбирает метод. После нажатия кнопки система встраивает ЦВЗ и показывает:

- изображение с ЦВЗ;
- `PSNR`;
- `SSIM`;
- `bpp`;
- `BER` без атаки;
- технические детали метода.

Эта вкладка полезна для технического сравнения, потому что на ней можно отдельно изучить устройство каждого алгоритма.

## 3. Пакетный benchmark

Это главный раздел для экспериментальной части.

Он принимает:

- много изображений через загрузку файлов;
- или путь к локальной папке с изображениями.

Далее система:

1. берет каждое изображение;
2. встраивает ЦВЗ выбранными методами;
3. применяет атаки из требований;
4. извлекает ЦВЗ после каждой атаки;
5. считает `PSNR`, `SSIM`, `bpp`, `BER`;
6. сохраняет CSV-отчет.

Колонки таблицы:

- `Изображение` — имя файла;
- `Метод` — `omniguard`, `lsb` или `dct`;
- `Атака` — примененная атака;
- `PSNR` — незаметность встраивания;
- `SSIM` — структурное сходство;
- `bpp` — емкость;
- `BER` — доля ошибочных бит после атаки;
- `Статус` — успешность обработки.

## 4. Сравнение со статьей

Эта вкладка нужна для сравнения изначальной и улучшенной логики локализации по метрикам, которые используются в статье OmniGuard.

Система:

1. встраивает ЦВЗ и payload;
2. применяет локальную атаку;
3. строит `ground_truth_mask`;
4. применяет выбранную глобальную деградацию;
5. анализирует результат изначальным режимом `watermark-only`;
6. анализирует тот же результат улучшенным режимом `hybrid`;
7. считает `Capacity`, `PSNR`, `SSIM`, `Bit Accuracy`, `BER`, `F1` и `AUC`.

Главное отличие от обычного benchmark: здесь обе ветки получают абсолютно одинаковые условия проверки.

Вкладка принимает одно изображение, несколько файлов или путь к папке. Для каждого изображения строится одинаковый эксперимент, а затем отдельно выводятся построчные метрики и средние значения по двум методам.
